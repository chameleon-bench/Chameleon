Overview

What is Chameleon?

The divergence between SQL dialects across different database systems poses challenges to aligning their semantic information through query translation, thereby impacting the reliability of database migration and cross-engine analytics.

Existing benchmarks for this task remain largely inherited from NL2SQL corpora, providing limited coverage of dialect-specific divergence and lacking both sufficient scale and ready-to-use cross-engine evaluation setups.

Chameleon is a large-scale benchmark for SQL dialect translation constructed through an end-to-end, divergence-driven synthesis framework. We mine 500+ structured dialect differences and decompose them into stage-specific requirements. Guided by these requirements, LLMs orchestrate a synthesis pipeline spanning schema construction, data population, and query synthesis, enabling the construction of equivalent multi-engine database instances with diverse executable queries.

Key Features

Why Chameleon?

📊

Scale & Diversity

1,200 cross-domain databases and over 100K translation queries distributed across four difficulty levels, compared with PARROT's 28K queries and DLBENCH's 6,402 tasks.

🎯

Divergence-Driven Coverage

Driven by over 500 structured dialect divergences mined from widely adopted transpilation engines. About 96.0% of valid generated queries exercise at least one dialect-specific feature.

⚡

Compact & Ready-to-Use

Databases average only 14 rows per table, and every database is materialized with equivalent schemas and identical data across all three engines, enabling direct cross-engine evaluation.

🔀

Standardized Splits

A standardized 1,000/100/100 train/dev/test split is provided, facilitating reproducible model development and fair comparison among translation systems.

🧠

Translation-Aware Difficulty

Four difficulty levels (easy, medium, hard, extra) defined by dialect divergence density and structural complexity, not generic query complexity.

✅

Execution-Validated

Every query is execution-validated on real database engines with an LLM-based self-repair loop, ensuring executability and meaningful result sets.
